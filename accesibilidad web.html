Investigación 3: Accesibilidad Web

1. ¿Qué es la Accesibilidad Web?

La Accesibilidad Web significa que los sitios web, herramientas y tecnologías están diseñados y desarrollados de tal manera que las personas con discapacidades pueden usarlos. Abarca todas las discapacidades que afectan al acceso a la Web, incluyendo las auditivas, cognitivas, neurológicas, físicas, del habla y visuales.

Importancia de la Accesibilidad

Justificación Ética

Desde un punto de vista ético, la accesibilidad es una cuestión de derechos humanos e inclusión social. La Convención de la ONU sobre los Derechos de las Personas con Discapacidad reconoce el acceso a la información, incluyendo la web, como un derecho humano. Excluir a las personas con discapacidad de este acceso es una forma de discriminación que limita su participación en la sociedad.

Justificación Legal

A nivel mundial, la accesibilidad web es cada vez más un requisito legal. Legislaciones como la Ley de Estadounidenses con Discapacidades (ADA) o el Acta Europea de Accesibilidad (EN 301 549) exigen que los sitios web, especialmente los de entidades públicas, sean accesibles. El incumplimiento puede resultar en demandas costosas y daño a la reputación.

Mejora de la Calidad del Código

La implementación de la accesibilidad conduce a un código de mayor calidad, más robusto y mantenible. Fomenta el uso de HTML semántico (lo que mejora el SEO), promueve la separación clara entre contenido (HTML), presentación (CSS) y comportamiento (JavaScript), y crea sitios más robustos que funcionan mejor en una gama más amplia de dispositivos y navegadores.

2. Principios de las WCAG (Pautas de Accesibilidad al Contenido en la Web)

Las WCAG son el estándar internacional para la accesibilidad web y se basan en cuatro principios fundamentales: perceptible, operable, comprensible y robusto (conocido por el acrónimo POUR).

1. Perceptible

La información y los componentes de la interfaz de usuario deben ser presentables a los usuarios de formas que puedan percibir. Esto significa que los usuarios deben poder percibir la información; no puede ser invisible a todos sus sentidos.

Ejemplo de Código (Texto Alternativo):

<!-- El atributo 'alt' permite que los lectores de pantalla describan la imagen -->
<img src="images/grafico.png" alt="Gráfico de barras que muestra un aumento del 30% en las ventas.">

2. Operable

Los componentes de la interfaz de usuario y la navegación deben ser operables. El usuario debe poder interactuar con la interfaz, asegurando que toda la funcionalidad esté disponible a través del teclado.

Ejemplo de Código (Botón Nativo):

<!-- Correcto: Operable con teclado y anunciado como 'botón' por defecto -->
<button onclick="enviarFormulario()">Enviar</button>

3. Comprensible

La información y el funcionamiento de la interfaz de usuario deben ser comprensibles. Los usuarios deben poder entender tanto la información como el funcionamiento de la interfaz.

Ejemplo de Código (Etiquetas de Formulario):

<!-- La etiqueta 'label' se asocia explícitamente con su campo 'input' -->
<label for="nombreUsuario">Nombre de usuario:</label>
<input type="text" id="nombreUsuario" name="username">

4. Robusto

El contenido debe ser lo suficientemente robusto como para que pueda ser interpretado de forma fiable por una amplia variedad de agentes de usuario, incluidas las tecnologías de asistencia.

Ejemplo de Código (HTML Bien Formado):

<!DOCTYPE html>
<html lang="es">
<head>
    <title>Página Robusta</title>
</head>
<body>
    <main>
        <h1>Título Principal</h1>
    </main>
</body>
</html>

3. Ejemplo Práctico: Implementación de Accesibilidad en este Sitio

Este sitio web de investigación ha sido desarrollado aplicando principios clave de accesibilidad para garantizar una experiencia inclusiva. A continuación, se detalla la implementación de tres requisitos fundamentales:

Navegación con el Teclado

Se ha garantizado que toda la funcionalidad del sitio sea completamente operable sin necesidad de un ratón. Todos los elementos interactivos, como los enlaces del menú de navegación, son accesibles mediante la tecla Tab para enfocarlos y la tecla Enter para activarlos. Esto se logra utilizando elementos HTML nativos (<a>) que incorporan esta funcionalidad por defecto.

HTML Semántico

La estructura de cada página está construida con un fuerte enfoque en la semántica para proporcionar un contexto claro a las tecnologías de asistencia. Se utilizan etiquetas como <header>, <nav>, <main>, <article> y <footer> para definir las principales regiones de la página, permitiendo a los usuarios de lectores de pantalla comprender la estructura del documento y navegar por él de manera eficiente.

WAI-ARIA (Web Accessibility Initiative – Accessible Rich Internet Applications)

Para mejorar la semántica donde el HTML nativo no es suficiente, se han utilizado atributos de WAI-ARIA. Un ejemplo claro es el menú de navegación, donde el enlace de la página actual incluye el atributo aria-current="page". Este atributo informa explícitamente a los usuarios de lectores de pantalla cuál es el enlace activo, proporcionando un contexto que de otro modo solo sería visible a través de estilos visuales.
